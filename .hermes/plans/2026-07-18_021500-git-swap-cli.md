# git-swap CLI Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task. Fresh subagent per task, two-stage review (spec compliance, then code quality).

**Goal:** Build a cross-platform CLI (`git-swap`) that lets a developer register multiple git accounts across multiple hubs (GitHub, GitLab, Azure DevOps, etc.) and instantly switch the active account — identity, commit-signing key, and SSH key — either globally or per-repository.

**Architecture:** A single static Go binary using Cobra for commands. Accounts ("profiles") are stored as YAML at `~/.config/git-swap/config.yaml`. All git/ssh mutations go through a small `Runner` interface (shells out to `git`/`ssh` in prod, faked in tests) so every command is unit-testable without touching the real system. Switching applies `user.name`, `user.email`, `user.signingkey`, `commit.gpgsign` via `git config` (global by default, `--local` for the current repo), and manages a git-swap–owned block in `~/.ssh/config` plus optional remote-URL rewriting for per-account SSH keys.

**Tech Stack:** Go 1.22+, `github.com/spf13/cobra` (CLI), `gopkg.in/yaml.v3` (config), Go stdlib `testing` (tests). No database. Git and SSH are driven by shelling out.

## Usage (target UX — what the finished CLI does)

```
git-swap — quickly switch between git accounts across multiple hubs

USAGE:
  git-swap <command> [flags]

COMMANDS:
  add <name>       Add or update an account profile
  list             List saved account profiles
  remove <name>    Remove an account profile
  use <name>       Switch the active account (global by default; --local for this repo)
  current          Show the active identity and which profile matches
  doctor           Check profiles for missing SSH keys / config issues
```

### Register accounts

git-swap supports two auth modes per account: `ssh` (default, uses a private key) and `https` (uses a Personal Access Token via git's credential helper).

```bash
# SSH account (default) — personal GitHub, key -> host alias github.com-personal
git-swap add personal \
  --hub github \
  --name "Jane Doe" \
  --email jane@personal.com \
  --ssh-key ~/.ssh/id_personal

# SSH work account on the SAME hub (this is why host aliases matter)
git-swap add work \
  --hub github \
  --name "Jane Doe" \
  --email jane@corp.com \
  --ssh-key ~/.ssh/id_work \
  --sign --signing-key ~/.ssh/id_work.pub

# HTTPS account — token pulled from an env var at switch time (never stored)
export GH_WORK_PAT=ghp_xxxxxxxx
git-swap add work-https \
  --hub github --auth https \
  --name "Jane Doe" --email jane@corp.com \
  --username jane-work --token-env GH_WORK_PAT

# HTTPS account — token read from a file instead
git-swap add gl-https \
  --hub gitlab --auth https \
  --email jane@personal.com \
  --username jane --token-file ~/.secrets/gitlab_pat

# Azure DevOps over SSH
git-swap add az-work --hub azure --email jane@corp.com --ssh-key ~/.ssh/id_azure
```

### Switch accounts

```bash
# SSH switch: identity + refresh the managed ~/.ssh/config block
git-swap use work
# switched to "work" (global, ssh) — Jane Doe <jane@corp.com>

# HTTPS switch: identity + set credential username + seed the PAT into git's helper
export GH_WORK_PAT=ghp_xxxxxxxx   # token must be resolvable at switch time
git-swap use work-https
# switched to "work-https" (global, https) — Jane Doe <jane@corp.com>

# Switch identity for the CURRENT repo only (leaves global untouched)
git-swap use personal --local
# switched to "personal" (local, ssh) — Jane Doe <jane@personal.com>

# Switch for this repo AND rewrite origin to the correct URL for the auth mode
git-swap use work --local --rewrite-remote acme/api
#   -> git@github.com-work:acme/api.git         (ssh alias)
git-swap use work-https --local --rewrite-remote acme/api
#   -> https://jane-work@github.com/acme/api.git (https w/ username)
```

### Check current state

```bash
git-swap current
# global: Jane Doe <jane@corp.com> -> profile work

git-swap current --local
# local: Jane Doe <jane@personal.com> -> profile personal

git-swap doctor
# personal         [ssh]   ok
# work             [ssh]   ok
# work-https       [https] ok (token resolvable)
# gl-https         [https] TOKEN unresolved: env var "GL_PAT" is empty or unset
```

### How multi-account-on-one-hub actually works

**SSH mode:** After `add`, git-swap owns a delimited block in `~/.ssh/config`:

```
# >>> git-swap managed block >>>
Host github.com-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_work
    IdentitiesOnly yes
Host github.com-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_personal
    IdentitiesOnly yes
# <<< git-swap managed block <<<
```

To use a specific SSH account for a repo, its remote must point at the alias host —
either clone with it (`git clone git@github.com-work:acme/api.git`) or let
`git-swap use <name> --local --rewrite-remote <owner/repo>` rewrite `origin` for you.

**HTTPS mode:** git can't tell two accounts on `github.com` apart by URL alone, so git-swap
sets a host-scoped credential username and seeds the PAT into git's credential helper:

```
git config --global credential.https://github.com.username jane-work
# + `git credential approve` fed:  protocol=https / host=github.com / username=jane-work / password=<PAT>
```

For per-repo HTTPS accounts, the remote should embed the username
(`https://jane-work@github.com/acme/api.git`) — which `--rewrite-remote` does for you —
so git looks up the matching stored credential. The PAT itself is resolved from
`--token-env`/`--token-file` at switch time and is never written to git-swap's config.

## Domain Notes (read before starting)

Switching a "git account" is really several independent concerns. This tool coordinates all of them behind one profile. Each profile picks an **auth mode** (`ssh` or `https`); identity and signing are shared, but the auth handling differs:

1. **Identity** — `git config user.name` / `user.email`. Global (`~/.gitconfig`) or local (`.git/config`). Same for both modes.
2. **Commit signing** — `user.signingkey`, `commit.gpgsign`, `gpg.format` (`ssh` or `openpgp`). Optional per profile. Same for both modes.
3a. **SSH auth** (`auth: ssh`) — different private keys per account. Standard trick: per-account Host aliases in `~/.ssh/config` (e.g. `github.com-work` → `IdentityFile ~/.ssh/id_work`). git-swap owns a managed block delimited by marker comments so it can rewrite safely without clobbering user entries.
3b. **HTTPS auth** (`auth: https`) — git authenticates HTTPS via a **credential helper** (macOS Keychain, `git-credential-store`, etc.), not keys. The account's password is a **Personal Access Token (PAT)**. Two GitHub accounts both on `github.com` collide unless disambiguated, so git-swap: (a) sets `credential.https://<host>.username = <user>` so git looks up the right credential, and (b) seeds the token non-interactively via `git credential approve` (piping `protocol/host/username/password`). The raw PAT is read from an env var (`--token-env`) or a file (`--token-file`) — never stored in git-swap's own config.
4. **Remote rewriting** — depends on auth mode. SSH: `origin` must point at the alias host (`git@github.com-work:org/repo.git`). HTTPS: `origin` embeds the username (`https://work-user@github.com/org/repo.git`) so the right credential is selected. `git-swap use --local <profile> --rewrite-remote <owner/repo>` produces the correct URL shape for the profile's mode.

**Profile fields:**
- `name` (unique key, e.g. `gh-personal`)
- `hub` (`github` | `gitlab` | `azure` | custom string — used for host defaults + labeling)
- `host` (e.g. `github.com`, `gitlab.com`; for SSH+azure `ssh.dev.azure.com`, for HTTPS+azure `dev.azure.com`)
- `auth` (`ssh` | `https`; default `ssh`)
- `git_name`, `git_email`
- `ssh_key` (path to private key — used when `auth: ssh`)
- `username` (HTTPS login used as the credential username — used when `auth: https`)
- `token_env` (name of env var holding the PAT — used when `auth: https`)
- `token_file` (path to a file holding the PAT; alternative to `token_env`)
- `signing_key` (optional), `sign` (bool)

**Auth-mode secret handling:** git-swap NEVER writes the PAT into `~/.config/git-swap/config.yaml`. At `use` time it resolves the token from `token_env` (preferred) or `token_file`, then hands it to git's credential helper via `git credential approve`. From then on git uses its own configured helper/keychain — git-swap does not persist the secret itself.

**Scope model:** `use` defaults to global. `--local` targets the current repo (must be inside a git work tree). `current` reports the effective identity for the current context.

```go
package run

import (
	"os/exec"
	"strings"
)

// Runner executes external commands. Prod uses Exec; tests use Fake.
// RunInput is like Run but pipes `stdin` to the command's standard input —
// used to hand secrets (PATs) to `git credential approve` without putting
// them on the argv (where they'd leak into process listings / Fake.Calls).
type Runner interface {
	Run(name string, args ...string) (string, error)
	RunInput(stdin string, name string, args ...string) (string, error)
}

type Exec struct{}

func (Exec) Run(name string, args ...string) (string, error) {
	out, err := exec.Command(name, args...).CombinedOutput()
	return strings.TrimSpace(string(out)), err
}

func (Exec) RunInput(stdin, name string, args ...string) (string, error) {
	cmd := exec.Command(name, args...)
	cmd.Stdin = strings.NewReader(stdin)
	out, err := cmd.CombinedOutput()
	return strings.TrimSpace(string(out)), err
}

type Fake struct {
	Outputs map[string]string // keyed by full "name arg arg" command line
	Errs    map[string]error
	Calls   []string
	Inputs  []string // stdin payloads passed to RunInput, in call order
}

func (f *Fake) Run(name string, args ...string) (string, error) {
	key := strings.TrimSpace(name + " " + strings.Join(args, " "))
	f.Calls = append(f.Calls, key)
	if f.Errs != nil {
		if e, ok := f.Errs[key]; ok {
			return "", e
		}
	}
	if f.Outputs != nil {
		if o, ok := f.Outputs[key]; ok {
			return o, nil
		}
	}
	return "", nil
}

func (f *Fake) RunInput(stdin, name string, args ...string) (string, error) {
	f.Inputs = append(f.Inputs, stdin)
	return f.Run(name, args...)
}
```

```go
package config

type Profile struct {
	Name       string `yaml:"name"`
	Hub        string `yaml:"hub"`         // github | gitlab | azure | custom
	Host       string `yaml:"host"`        // github.com, gitlab.com, ssh.dev.azure.com
	Auth       string `yaml:"auth,omitempty"` // ssh | https (default ssh)
	GitName    string `yaml:"git_name"`
	GitEmail   string `yaml:"git_email"`
	SSHKey     string `yaml:"ssh_key,omitempty"`   // used when Auth==ssh
	Username   string `yaml:"username,omitempty"`  // HTTPS credential username (Auth==https)
	TokenEnv   string `yaml:"token_env,omitempty"` // env var holding PAT (Auth==https)
	TokenFile  string `yaml:"token_file,omitempty"`// file holding PAT (Auth==https)
	SigningKey string `yaml:"signing_key,omitempty"`
	Sign       bool   `yaml:"sign,omitempty"`
}

// AuthMode returns the effective auth mode, defaulting to "ssh".
func (p Profile) AuthMode() string {
	if p.Auth == "https" {
		return "https"
	}
	return "ssh"
}

type Config struct {
	Profiles []Profile `yaml:"profiles"`
}

func (c *Config) Find(name string) (Profile, bool) {
	for _, p := range c.Profiles {
		if p.Name == name {
			return p, true
		}
	}
	return Profile{}, false
}

func (c *Config) Upsert(p Profile) {
	for i := range c.Profiles {
		if c.Profiles[i].Name == p.Name {
			c.Profiles[i] = p
			return
		}
	}
	c.Profiles = append(c.Profiles, p)
}

func (c *Config) Remove(name string) bool {
	for i := range c.Profiles {
		if c.Profiles[i].Name == name {
			c.Profiles = append(c.Profiles[:i], c.Profiles[i+1:]...)
			return true
		}
	}
	return false
}
```

```go
import (
	"os"
	"path/filepath"

	"gopkg.in/yaml.v3"
)

// DefaultPath returns ~/.config/git-swap/config.yaml (honors XDG_CONFIG_HOME).
func DefaultPath() (string, error) {
	base := os.Getenv("XDG_CONFIG_HOME")
	if base == "" {
		home, err := os.UserHomeDir()
		if err != nil {
			return "", err
		}
		base = filepath.Join(home, ".config")
	}
	return filepath.Join(base, "git-swap", "config.yaml"), nil
}

func Load(path string) (*Config, error) {
	data, err := os.ReadFile(path)
	if os.IsNotExist(err) {
		return &Config{}, nil
	}
	if err != nil {
		return nil, err
	}
	var c Config
	if err := yaml.Unmarshal(data, &c); err != nil {
		return nil, err
	}
	return &c, nil
}

func Save(path string, c *Config) error {
	if err := os.MkdirAll(filepath.Dir(path), 0o755); err != nil {
		return err
	}
	data, err := yaml.Marshal(c)
	if err != nil {
		return err
	}
	return os.WriteFile(path, data, 0o600)
}
```

```go
package gitcfg

import (
	"github.com/jokot/git-swap/internal/config"
	"github.com/jokot/git-swap/internal/run"
)

// Apply writes identity and signing settings for the profile.
// scopeLocal=true targets the current repo (.git/config); otherwise global.
func Apply(r run.Runner, p config.Profile, scopeLocal bool) error {
	scope := "--global"
	if scopeLocal {
		scope = "--local"
	}
	sets := [][2]string{
		{"user.name", p.GitName},
		{"user.email", p.GitEmail},
	}
	if p.Sign {
		if p.SigningKey != "" {
			sets = append(sets, [2]string{"user.signingkey", p.SigningKey})
		}
		sets = append(sets, [2]string{"commit.gpgsign", "true"})
	}
	for _, kv := range sets {
		if _, err := r.Run("git", "config", scope, kv[0], kv[1]); err != nil {
			return err
		}
	}
	return nil
}

// Current reads the effective identity for the given scope.
func Current(r run.Runner, scopeLocal bool) (name, email string) {
	scope := "--global"
	if scopeLocal {
```

```go
	endMarker   = "# <<< git-swap managed block <<<"

// AliasFor returns the SSH Host alias for a profile, e.g. "github.com-work".
func AliasFor(p config.Profile) string {

// RenderBlock returns a marker-delimited SSH config block for all profiles
// that declare an SSHKey. Profiles without a key are skipped.
func RenderBlock(profiles []config.Profile) string {
	var b strings.Builder
	b.WriteString(beginMarker + "\n")

		b.WriteString(fmt.Sprintf("    IdentityFile %s\n", p.SSHKey))
		b.WriteString("    IdentitiesOnly yes\n")
	}
	b.WriteString(endMarker + "\n")
	return b.String()
```

```go
// Splice replaces an existing managed block in `existing` with `block`,
// or appends `block` if no managed block is present. User content is preserved.
func Splice(existing, block string) string {
	start := strings.Index(existing, beginMarker)
	end := strings.Index(existing, endMarker)
	if start != -1 && end != -1 && end > start {
		end += len(endMarker)
		// consume a trailing newline after the end marker if present
		tail := existing[end:]
		tail = strings.TrimPrefix(tail, "\n")
		before := existing[:start]
		return before + strings.TrimSuffix(block, "\n") + "\n" + tail
	}
	sep := ""
	if existing != "" && !strings.HasSuffix(existing, "\n") {
		sep = "\n"
	}
	if existing != "" {
		sep += "\n"
	}
	return existing + sep + block
}
```

```go
// SyncFile reads the ssh config at path (if any), splices in the managed block
// for the given profiles, and writes the result with 0600 perms.
func SyncFile(path string, profiles []config.Profile) error {
	if err := os.MkdirAll(filepath.Dir(path), 0o700); err != nil {
```

```go
// ResolveToken returns the PAT for an HTTPS profile, read from the env var

func ResolveToken(p config.Profile) (string, error) {
	if p.TokenEnv != "" {
		if v := os.Getenv(p.TokenEnv); v != "" {

		}
		return strings.TrimSpace(string(data)), nil
	}
	return "", fmt.Errorf("profile %q has no token_env or token_file", p.Name)
}
```

```go
// Apply configures the HTTPS credential username for the profile's host and
```

```go
package cmd

import (
	"fmt"

	"github.com/jokot/git-swap/internal/gitcfg"
	"github.com/jokot/git-swap/internal/httpcfg"
	"github.com/jokot/git-swap/internal/sshcfg"
	"github.com/spf13/cobra"
)

func newUseCmd(app *App) *cobra.Command {
	var local bool
	var rewriteRemote string
	c := &cobra.Command{
		Use:   "use <name>",
		Short: "Switch the active git account (global by default, --local for this repo)",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			cfg, err := app.load()
			if err != nil {
				return err
			}
			p, ok := cfg.Find(args[0])
			if !ok {
				return fmt.Errorf("profile %q not found", args[0])
			}
			if err := gitcfg.Apply(app.Runner, p, local); err != nil {
				return fmt.Errorf("apply git config: %w", err)
			}

			switch p.AuthMode() {
			case "ssh":
				if err := sshcfg.SyncFile(app.SSHPath, cfg.Profiles); err != nil {
					return fmt.Errorf("sync ssh config: %w", err)
				}
			case "https":
				// Set credential username (disambiguates same-host accounts) and
				// seed the PAT into git's credential helper via stdin.
				if err := httpcfg.Apply(app.Runner, p, local); err != nil {
					return fmt.Errorf("apply https credentials: %w", err)
				}
			}

			if rewriteRemote != "" {
				url := remoteURL(p, rewriteRemote)
				if url == "" {
					return fmt.Errorf("cannot build remote for profile %q", p.Name)
				}
				if _, err := app.Runner.Run("git", "remote", "set-url", "origin", url); err != nil {
					return fmt.Errorf("rewrite remote: %w", err)
				}
			}
			scope := "global"
			if local {
				scope = "local"
			}
			fmt.Fprintf(cmd.OutOrStdout(), "switched to %q (%s, %s) — %s <%s>\n", p.Name, scope, p.AuthMode(), p.GitName, p.GitEmail)
			return nil
		},
	}
	c.Flags().BoolVar(&local, "local", false, "apply to the current repository only")
	c.Flags().StringVar(&rewriteRemote, "rewrite-remote", "", "rewrite origin for <owner/repo> using this profile's auth mode")
	return c
}

// remoteURL builds the origin URL for a profile+repo per its auth mode.
//   ssh:   git@<host>-<name>:<owner/repo>.git
//   https: https://<username>@<host>/<owner/repo>.git
func remoteURL(p config.Profile, ownerRepo string) string {
	switch p.AuthMode() {
	case "ssh":
		if p.SSHKey == "" {
			return ""
		}
		return fmt.Sprintf("git@%s:%s.git", sshcfg.AliasFor(p), ownerRepo)
	case "https":
		if p.Username == "" {
			return ""
		}
		return fmt.Sprintf("https://%s@%s/%s.git", p.Username, p.Host, ownerRepo)
	default:
		return ""
	}
}
```

```go
	"github.com/jokot/git-swap/internal/httpcfg"
	"github.com/spf13/cobra"

func newDoctorCmd(app *App) *cobra.Command {
	return &cobra.Command{
		Use:   "doctor",
		Short: "Check profiles for missing SSH keys / unresolvable tokens",
		Args:  cobra.NoArgs,
		RunE: func(cmd *cobra.Command, args []string) error {
			cfg, err := app.load()
			if err != nil {
				return err
			}
			out := cmd.OutOrStdout()
			if len(cfg.Profiles) == 0 {
				fmt.Fprintln(out, "no profiles configured")
				return nil
			}
			for _, p := range cfg.Profiles {
				status := "ok"
				switch p.AuthMode() {
				case "ssh":
					if p.SSHKey != "" {
						path := p.SSHKey
						if strings.HasPrefix(path, "~/") {
							home, _ := os.UserHomeDir()
							path = filepath.Join(home, path[2:])
						}
						if _, err := os.Stat(path); err != nil {
							status = "MISSING ssh key: " + p.SSHKey
						}
					} else {
						status = "no ssh_key set"
					}
				case "https":
					if _, err := httpcfg.ResolveToken(p); err != nil {
						status = "TOKEN unresolved: " + err.Error()
					} else {
						status = "ok (token resolvable)"
					}
				}
				fmt.Fprintf(out, "%-16s [%s] %s\n", p.Name, p.AuthMode(), status)
			}
			return nil
		},
	}
}
```

## Risks, Tradeoffs & Open Questions

1. **PAT storage & security.** git-swap never writes the token into its own `config.yaml` — it stores only `token_env`/`token_file` and resolves the secret at `use` time, handing it to git's credential helper via `git credential approve` (piped on stdin, never argv). BUT the security of the token at rest then depends on git's configured helper: on macOS `osxkeychain` is secure; the plain `store` helper writes `~/.git-credentials` in cleartext. Open question: should `doctor` warn when the active `credential.helper` is `store`? (Recommended follow-up.)
2. **HTTPS requires a credential helper.** Seeding via `git credential approve` only persists if a helper is configured (`git config --global credential.helper osxkeychain|store|manager`). If none is set, git won't remember the token across invocations. `doctor`/README should surface this. The smoke test sets `store` explicitly for this reason.
3. **Azure DevOps quirks.** Azure differs by mode: SSH uses `ssh.dev.azure.com` with `v3/org/project/repo` paths; HTTPS uses `dev.azure.com/org/project/_git/repo`. `--rewrite-remote` takes the `owner/repo`-style value verbatim and does not synthesize Azure's special path shapes. Open question: add hub-specific URL templating later?
4. **Global vs local footgun.** `use` defaults to global, which changes machine-wide identity and (for https) the global credential username. Some prefer `--local` as the default. (Current plan: global default, documented clearly.)
5. **SSH agent / IdentitiesOnly.** The managed block sets `IdentitiesOnly yes` so the correct key is used deterministically; users relying on agent forwarding may need to adjust. Documented in README.
6. **Module path.** `github.com/jokot/git-swap` is assumed — change in Task 0.1 if the repo will live elsewhere. All internal imports must match.
7. **Signing format.** Plan sets `commit.gpgsign` + `user.signingkey` but does not set `gpg.format`. If a profile uses SSH signing, add `gpg.format=ssh`. Consider a `signing_format` profile field in a follow-up.
